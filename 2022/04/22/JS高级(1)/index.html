Level 1

Array

new Array()

调用构造函数 new Array() 时，若传入的是一个数值，则会创建一个指定长度的数组，如果是其他类型的值，则会创建一个只包含该特定值的数组

from() 和 of()

es6新增创建数组的两个静态方法 from() 和 of()
from() 用于将数组结构转化为数组实例，of() 用于将一组参数转化为数组实例
Array.from() 的第一个参数是一个类数组对象，即任何可迭代的结构，或者一个有length属性和可索引元素的结构

Array.from() 第二个可选参数是映射函数，可以直接增强新数组的值，第三个可选的参数用于指定映射函数的this值，但是这个重写的this在箭头函数中不适用

Array.of()

数组空位

使用数组字面量创建数组时，可以用一串逗号来创建空位,es6普遍将这些空位当作存在的元素，只不过值为undefined

注意： 数组空位存在隐患，尽量不要使用，可以显式的用undefined代替

数组索引

js中数组length值的特殊之处在于他不是只可读，通过修改length值可以在数组末尾删除或添加元素

迭代器方法

es6新增三个用于检测数组内容的方法: keys()返回数组索引的迭代器， values()返回数组元素的迭代器，entries()返回索引/值对的迭代器

排序方法

数组有两个方法可以对数组元素排序 sort() 和 reverse()
默认情况下sort()会升序排列数组，sort()会在每一项上都调用String()转型函数，然后比较字符串来进行排列，即使元素是数值

reverse()反向排列元素

操作方法

concat()方法可以在现有数组的基础上创建一个副本，并将他的参数添加到副本末尾。不会改变原数组

在参数数组上指定一个特殊的参数 Symbol.isConcatSpreadable ,这个符号可以选择是否打平参数数组 将Symbol.isConcatSpreadable设置为true可以强制打平数组，false可以不打平数组

slice()方法用于创建包含原数组一个或多个元素的副本，接收两个参数，第一个参数是开始索引，第二个是结束索引，如果只有一个参数，即返回该参数到数组最后一个元素，如果参数为负数，会将他转化成 数组长度 加上 这个负数参数

splice()方法或许是最强大的数组方法，他的主要目的是在数组中间插入元素，直接操作原数组
删除： 传入两个参数，要删除的第一个元素的位置 和 要删除的元素数量
插入： 传入三个参数，开始位置，0 (要删除的元素数量)，要插入的元素(可以多个)
替换： 传入三个参数，开始位置，要删除的元素数量，要插入的元素(可以多个)

迭代方法

每个方法接受两个参数：以每一项为参数运行的函数，以及可选的作为函数运行上下文的作用域对象 (影响函数中this的值)， 传给每个方法的函数接受三个参数：数组元素，元素索引，数组本身

方法 描述
every() 对数组的每一项都运行传入的函数，如果对每一项返回的都是true，则这个方法返回true
filter() 对数组的每一项都运行传入的函数，函数返回true的项会组成数组之后返回
forEach() 对数组的每一项都运行传入的函数，无返回值
map() 对数组的每一项都运行传入的函数，返回由每次函数调用的结果组成的数组
some() 对数组的每一项都运行传入的函数，如果有一项返回的是true，则这个方法返回true
这些方法都不改变调用他们的数组

归并方法

reduce()和reduceRight(),这两个方法会迭代数组的每一项，并在此基础返回一个迭代好的最终值，reduce()是从数组的第一项开始迭代到最后，reduceRight()正好相反
这两个函数都有两个参数：

  1. 每一项都会执行的函数(这个函数有四个参数：上一次的归并返回值，当前项，当前项索引，数组本身，这个函数的归并返回值会作为下一次调用此函数的第一个参数)
  2. 归并起始值(可选)

控制台打印结果：因为我们给reduce传入了第二个参数(5)，所以它的起始值就是5，在迭代了所有项时，最后sum的结果为20

Map

在ES6之前，JS中实现“键/值”式存储可以使用Object来高效的完成。在ES6中新增了Map，Map是一种新的集合类型，为JS带来了真正的键值对存储机制

基本API

是由new关键字和Map构造函数来创建一个实例

如果要在创建实例的同时初始化实例，可以给他传入一个可迭代的对象，需要包含键值对数组

初始化实例之后，可以通过set()方法来添加键值对，通过has()和get()方法来进行查询，通过size属性来获取映射中的键值对数量，通过delete()和clear()方法来删除值

set()方法会返回Map实例，所以可以把多个操作连缀起来，包括初始化声明

与Object只能使用数值、字符串或符号作为键不同，Map可以使用任何JavaScript数据类型作为键

顺序与迭代

与Object的一个主要差异是，Map会维护键值对的插入顺序，因此可以按照插入顺序进行迭代操作

映射实例提供了一个迭代器(iterator)，能以插入顺序生成[key, value]形式的数值，可以通过entries()方法(或者Symbol.iterator属性，它引用了entries())来得到这个迭代器

keys()和values()方法分别返回以插入顺序生成键和值的迭代器

键和值在迭代器遍历是是可以修改得，但是映射内部的引用则无法修改

但是可以修改键或值的对象内部的属性，因为这样不影响他们在映射中的身份

Object和Map的比较

  1. 占用内存
    Object和Map存储单个键值对所占用的内存空间都随着键的数量线性增加，批量删除或添加键值对则取决于不同浏览器对该类型的内存分配。但给定内存大小，Map大约可以比Object多存储50%的键值对
  2. 插入性能
    向Object和Map中插入新键值对消耗大致相同，不过Map插入在所有浏览器中会稍微快一点。如果代码涉及大量数据插入操作，明显Map性能占优
  3. 查找速度
    大型Object和Map的查找速度性能差异极小，但如果只有少量数据时，Object速度会更快一点
  4. 删除性能
    使用delete删除Object属性的性能一直饱受诟病，Map的delete()操作比查找和插入都更快，如果代码涉及大量数据删除，毫无疑问选择Map

Set

ES6新增的Set是一种新集合类型，为这门语言带来集合数据结构，Set在很多方面都像是加强版的Map，因为他们的大多数API和行为都是共有的

基本API

使用new关键字和Set构造函数可以创建一个空集合

如果要在创建实例的同时初始化实例，可以给他传入一个可迭代的对象，需要包含插入到新集合实例中的元素

初始化之后，使用add()增加值，使用has()查询值，通过size属性获取元素数量，以及使用delete() clear()删除值

add()方法会返回Set实例，所以可以把多个操作连缀起来，包括初始化声明

与Map类似，Set可以包含任何Javascript数据类型作为值

顺序与迭代

Set会维护值插入的顺序，因此支持按顺序迭代
集合实例可以提供一个迭代器，能以插入顺序生成集合内容。可以通过values()方法及其别名方法keys()( 或者Symbol.iterator属性，它引用values())来取得这个迭代器

集合的entries()方法返回一个迭代器，可以按照插入顺序产生包含两个元素的数组，这两个元素是集合每个值的重复出现

修改集合中值的属性不会影响其作为集合值的身份，且字符串原始值作为值不会被修改

函数

函数实际上就是对象，每个函数都是Function类型的实例。而Function也有属性和方法，跟其他的引用类型一样。因为函数是对象，所以函数名就是指向函数对象的指针
函数通常以函数声明的方式定义

函数表达式的方式定义

箭头函数的方式定义

箭头函数

ES6新增使用=>语法定义函数表达式，箭头函数实例化的函数对象与正式的函数表达式创建的函数对象行为相同

如果只有一个参数，可以不用写括号。只有没有参数，或者多个参数的情况下才需要写括号

箭头函数也可以不用大括号，但这样会改变函数的行为。使用大括号说明包含‘函数体’，可以在一个函数中包含多条语句，跟常规的函数一样。如果不使用大括号，那么箭头后面就只能有一行代码，比如一个赋值，一个表达式。而且，省略大括号会隐式返回这行代码的值

箭头函数虽然简洁，但也有许多场合不适用，箭头函数不能使用argument、super和new.target，也不能作为构造函数。此外，箭头函数没有prototype属性

函数名

因为函数名就是指向函数的指针，所以它们跟其他包含对象指针的变量具有相同的行为，这就意味着一个函数可以有多个名称

ES6的所有函数对象都会暴露一个只读的name属性，其中包含函数的信息，如果函数没有名称，就会如实显示成空字符串，如果他是使用Function构造函数创建的，则会被标识为’anonymous’

理解参数

ECMAScript函数的参数和大多数其他语言不同，ECMAScript函数既不关心关心传入参数的个数，也不关心传入参数的类型。定义函数时要接收两个参数，并不意味着你一定要传两个参数，你可以传一个，三个，甚至一个也不传，都不会报错

其实，ECMAScript函数的参数在内部表现为一个数组，函数被调用时总会接收一个数组。在使用function关键字定义(非箭头)函数时，可以在函数内部访问arguments对象，从中取得传进来的每个参数

arguments对象是一个类数组对象(不是Array实例)，因此可以使用中括号语法访问其中的元素(例如：arguments[0])，可以用arguments.length来确实传入的参数个数

注意：如果函数是使用箭头语法定义的，那么传给函数的参数就不能使用arguments关键字访问，只能通过定义的命名参数访问

虽然箭头函数没有arguments属性，但可以在包装函数中把他提供给箭头函数

没有重载

ECMAScript函数不能像传统编程那样重载，如前所述，函数的参数是由零个或多个值组成的数组，没有办法进行重载
在ECMAScript中，如果先后定义两个同名函数，后定义的会覆盖前一个

当然可以通过检测传入的参数的类型和数量，然后分别执行不同的逻辑来模拟函数重载

在前面说过，函数名相当于是指向函数的指针，那么就很好理解，当定义第二个同名函数时，函数名指向了第二次定义的那个函数

每个函数都有两个属性：length和prototype

call()接收的第一个参数也是指定的this值，之后的参数是要逐个传递

使用apply和call的好处就是可以将 任意对象 设置为 任意函数的作用域

BOM

BOM(浏览器对象模型， Browser Object Model)

window对象

BOM的核心是window对象，表示浏览器的实例。window对象在浏览器中有两重身份，一个是ECMAScript的Global对象，另一个是浏览器窗口的JavaScript接口。网页中定义的所有变量、对象和属性都以window作为其Global对象。

因为window对象的属性在全局作用域中有效，所以许多浏览器API都以window对象属性的形式暴露出来

Global作用域

因为window对象被复用为ECMAScript的Global对象，所以任何用var声明的变量和函数都会变成window的属性和方法

窗口大小

可以通过innerWidth，innerHeight，outerHeight，outerWidth来得到窗口大小
outerHeight和outerWidth返回浏览器窗口自身的大小
innerWidth和innerHeight返回浏览器窗口中页面视口的大小(不包含浏览器边框和工具栏)

同时可以通过document.documentElement.clientWidth和document.documentElement.clientHeight返回页面视口的宽度和高度

视口位置

有时候浏览器无法将整个页面装下，所以需要拖动滚动条来查看。

可以使用window.pageXoffset / window.pageYoffset和window.scrollX / window.scrollY来得到当前位置相对于视口滚动的距离

可以通过scroll()、scrollTo()和scrollBy()来实现页面的滚动，这三个方法都接收表示相对视口距离的x和y，在两个参数在前两个方法中都表示要滚动到的坐标，在最后一个方法中表示滚动的距离

定时器

JavaScript在浏览器是单线程执行的，但是可以通过定时器在指定的某个时间之后或者每隔多久执行相应的代码。setTimeout()用于在指定时间之后执行某些代码，setInterval()用于在每隔一段时间执行某些代码

setTimeout接收两个参数：要执行的代码 和 在执行回调函数前等待的时间(毫秒)

注意： 第二个参数是等待的毫秒数，不是执行代码的确切时间。因为JavaScript是单线程的，所以每次只能执行一段代码。为了调度不同代码的执行，JS维护了一个任务队列，其中的任务会按照添加的前后顺序执行。setTimeout的第二个参数只是告诉JS在指定时间后将他加入到这个队列中。如果队列是空，那立刻执行这些代码；如果队列内还有其他内容，则代码必须等前面的任务执行玩完才能执行

调用setTimeout时，会返回一个ID，这个ID是这个setTimeout的唯一标识符，可用于取消该任务。当这个定时器还没有被执行前，可以调用clearTimeout()并传入ID来取消该定时器

setInterval()和setTimeout()的使用方式相似，只不过指定的内容每隔指定时间就会被调用，直到取消定时器或者页面销毁，setInterval的第二个参数是每次的间隔时间

同样的setInterval()也返回一个ID，可以通过clearInterval()来取消定时器

DOM

DOM(文档对象模型， Document Object Model)
DOM表示由多层节点构成的文档，通过它开发者可以添加、删除和修改页面的各个部分

上图就是所示代码的节点结构
document表示每个文档的根节点，根节点的唯一子节点是<html>元素，我们称之为文档元素。
文档元素是整个文档最外层的元素，所有的其他元素都存在于这个文档元素之内

Node类型

在JS中，所有节点类型都继承Node类型，因此所有类型都共享相同的基本属性和方法

每个节点都有nodeType属性，表示该节点的类型。共有12个节点类型，用数值常量来表示
比较常见的的三个节点类型

  1. Node.ELEMENT_NODE 对应的数值常量 1
  2. Node.TEXT_NODE 对应的数值常量 3
  3. Node.DOCUMENT_NODE 对应的数值常量 9

节点类型可以通过与常量的比较来确定

节点关系

文档中的所有节点都和其他节点有关系，这就形成了一个DOM树

每一个节点都有一个childNodes属性，它包含一个NodeList实例。NodeList是一个类数组对象，它有序的存储着节点.(虽然nodeList不是Array实例，但是可以使用中括号访问他的值，也有length属性)

可以使用中括号或item()访问他的值

每个节点都有parentNode属性，指向其DOM树中的父节点。
childNodes中的所有节点都有一个共同的父节点(parentNode)。
childNodes中的每个节点都可以通过previousSibling(前一个兄弟节点)和nextSibling(下一个兄弟节点)来访问他的兄弟节点
childNodes列表中第一个节点的previousSibling属性是null；最后一个节点的nextSibling属性是null
firstChild指向childNodes中的第一个节点；lastchild指向childNodes中的最后一个节点

hasChildNodes()方法可以检测节点是否有子节点，有子节点则返回true，没有则返回false

操作节点

使用appendChild()向childNodes列表末尾添加节点，当使用appendChild添加以存在childNodes列表中的节点时，会将这个节点从原来的位置移动到childNodes列表的末尾

使用insertBefore()可以向childNodes任意位置插入节点，它接收两个参数：要插入的节点 和 参照节点
insertBefore会将要插入的节点插入到参照节点之前，并返回要插入的节点。
如果参照节点是null，insertBefore会将节点插入到childNodes列表的末尾，和appendChild的功能一致

前面两种方法都没有删除节点，相对地，replaceChild()接收两个参数：要插入的节点 和 要替换的节点。被替换的节点会从DOM中完全移除

使用removeChild()来移除节点，它接收一个参数，就是要移除的节点。被移除的节点会被返回

使用cloneNode()方法，可以返回与调用它的节点一模一样的节点。cloneNode接收一个布尔值参数，表示是否深拷贝。传入true时，会进行深拷贝，会复制节点以及它的整个子DOM树。传入false时，进行浅拷贝，只复制调用该方法的节点

Document类型

Document类型是JS中表示文档类型的节点。在浏览器中，文档对象document 是 HTMLDocument的实例（HTMLDocument继承Document）

文档子节点

document.documentElement可以访问文档子节点，它始终指向HTML页面中的<html>元素
document.body可以直接访问<body>元素

定位元素

getElementById()方法接收一个参数，就是要获取的元素的id。如果找到就返回这个元素，否则返回null。
当页面中有多个id相同的元素，那么这个方法会返回找到的第一个元素

getElementsByTagName()方法接收一个参数，就是要获取的标签名，返回包含零个或多个的NodeList

getElementsByName()方法接收一个参数，就是要获取的给定name属性值的元素。多用于单选按钮

Element类型

Element表示XML或者HTML元素，对外暴露访问元素标签名，子节点和属性的能力

HTML元素

所有HTML元素都通过HTMLElement类型表示，HTMLElement继承Element并增加了一些属性
所有HTML元素上都有的属性：

  1. id 元素在文档中的唯一标识
  2. title 包含元素的额外信息
  3. lang 元素内容的语言代码（很少使用）
  4. dir 语言的书写方向（”ltr”表示从左到右，”rtl”表示从右到左，同样很少使用）
  5. className 相当于class属性（class是ECNAScript的关键字，所以不能直接使用class这个名字）

创建元素

事件捕获

Netscape提出了事件捕获的事件流。事件捕获的意思是最不具体的节点应该最先收到事件，最具体的节点应该最后收到事件。事件捕获是为了事件到达最终目标前拦截事件。使用前面的代码来说明事件捕获的顺序：document -> <html> -> <body> -> <div>

事件类型

比较常用的几个事件：
用户界面事件（UIEvent）

  1. load 在window上页面加载完成时触发，在<img>上图片加载完成时触发，在窗套<frameset>上当所有窗格<frame>加载完成时触发，在<object>元素上相应对象加载完成时
  2. unload 上述东西被卸载完成后触发

焦点事件

  1. blur 当元素失去焦点时触发，不冒泡
  2. focus 当元素取得焦点时触发，不冒泡
  3. focusin 元素取得焦点时触发，冒泡
  4. focusout 元素失去焦点触发，冒泡

鼠标滚轮事件

  1. click 单击鼠标左键或者按下键盘回车键
  2. dblclick 双击左键
  3. mousedown 按下任意鼠标键
  4. mouseenter 鼠标从外部移动到内部
  5. mouseleave 鼠标从内部移动到外部
  6. mouseup 鼠标键释放时

键盘与输入事件

  1. keydown 按下某个键盘上的键，持续按住会重复触发
  2. keyup 释放某个键

在keyup或keydown事件触发时，event对象都会保存一个keyCode属性，它表示键码，它的值与小写字母和数字的asc码一致
