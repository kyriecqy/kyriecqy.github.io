现代JavaScript

JS基础知识

数据类型

JavaScript中有8种基本的数据类型（7种原始类型和1种引用类型）

它们分别是：number bigInt string boolean null undefined object symbol

number

number代表整数和浮点数，范围 -(2^53 - 1) 到 (2^53 - 1)

还有三个比较特殊的数值

  1. Infinity 表示无穷大
  2. -Infinity 表示负无穷大
  3. NaN 代表一个计算错误，对NaN的任何进一步数学运算都是NaN

bigInt

表示超出Number类型精度之外的整数
通过在整数字段末尾添加n来表示

null

null独立构成一个类型，只包含null

JavaScript中的null表示一个”无”、”空”或”值未知”的特殊值

undefined

undefined也是自成一个类型

它表示 未被赋值

如果一个变量被声明，但是未赋值，那么它的值就是undefined

值的比较

不同类型间的比较

对于不同类型值的比较，JS会首先尝试将其转化为数字（number）再判定大小

对null和undefined进行比较

显然，在严格模式下===，因为它们属于不同的类型，所以不相等

在非严格模式下==，两者是相等的

另外，在使用数学式或者其他比较方法< > <= >=时
null会被转化为 0
undefined会被转化为 NaN

null的示例说明：null与0的比较

对于(1)、(3)来说，在进行> >=比较时，null被转化成0，显然null>=0
对于(2)来说，null和undefined在进行相等性==比较时，不会进行类型转化。所以在非严格模式下，除了它们之间互等，不与其他相等

undefined的示例说明：undefined不应该与其他值进行比较

逻辑运算符

JS中的假值；undefined、null、0、NaN、””（空字符串，不是空格字符串）、false

除了上述的7种以外都是真值

||（或）

大部分情况下，逻辑或||用来测试是否有任何给定的条件为true

在JS中，逻辑或||的用法更加灵活，它会将不是布尔值的操作数转化为布尔值来参与运算

或运算寻找第一个真值

给定多个参与或运算的值

||做了如下的事情：

  1. 从左到右依次计算操作数
  2. 处理每一个操作数时，都将其转化为布尔值。如果结果是true，就停止计算返回这个操作数的初始值
  3. 如果所有的操作数都被计算过（且结果都是false）。就返回最后一个操作数的初始值

简单地说：||操作符返回遇到的第一个真值，如果不存在真值，就返回最后一个值

&&（与）

在传统编程中，当多个操作数都是真值时，返回true。否则返回false

与或运算符一样，在JS中它也有一些灵活的用法

与运算寻找第一个假值

&&做了如下的事情：

  1. 从左到右依次计算操作数
  2. 处理每一个操作数时，都将其转化为布尔值。如果结果是false，就停止计算返回这个操作数的初始值
  3. 如果所有的操作数都被计算过（且结果都是true）。就返回最后一个操作数的初始值

函数进阶

执行上下文和堆栈

我们现在有一个十分常见的递归函数

下面来研究一下递归调用是如何运行的

正在运行的函数的执行过程的相关信息会被存储在执行上下文中

执行上下文是一个内部数据结构，它包含有关函数执行的详细细节：当前控制流所在位置、当前变量、this值等

当一个函数被嵌套调用时，会发生一下的事：

  1. 代码执行到嵌套调用处，当前函数被暂停
  2. 与它关联的执行上下文被一个叫做执行上下文堆栈的特殊数据结构保存
  3. 执行嵌套调用
  4. 嵌套调用结束后，从堆栈中恢复之前的执行上下文，并从停止的位置恢复外部函数

防抖装饰器

防抖在实际中经常被用到，例如用户要输入一段内容，在输入完成之后向服务器发送请求。
显然我们没必要用户每输入一个字符就发送一次请求。相反我们需要等待一段时间确认用户输入完毕，再发起请求
又比如我们有一个按钮来发送请求，当请求响应比较慢时，用户可能会频繁点击按钮，这样会额外多发送许多请求到服务器，增加了服务器的压力

节流装饰器

节流装饰器与防抖装饰器不同点在于：
防抖指的是：例如如果用户一直在输入则一直不发送请求，等到用户输入完毕再等相应时间后发送请求
节流指的是：在规定的时间内都会发送一次请求

  1. 在第一次调用期间，wrapper只运行func并设置冷却状态
  2. 在这种状态下，所有的调用都被记录在savaArgs savaThis中
  3. 经过ms毫秒之后，冷却状态解除，最后一次被记录的savaArgs savaThis会被使用

Promise

语法：

传递给new Promise()的函数称为executor，它有两个参数resolve和reject，它们是JS自身提供的回调。
当executor获得了结果，它应该调用以下回调之一 ：

  1. resolve(value) executor执行过程中没有出现错误，返回结果value
  2. reject(err) executor执行过程中发生了错误，返回错误信息

也就是说，executor会自动运行并执行一些代码，如果成功则调用resolve，发生错误则调用reject

new Promise返回的对象有一些内置属性：

  1. state 最初是pending，在resolve被调用后变成fulfilled，在reject被调用后变成rejected
  2. result 最初是undefined，当resolve(value)被调用后变成value，当reject(error)被调用后变成error

executor只能调用一个resolve或reject，一般调用了其中一个，之后的更改都无效

Promise链的想法就是.then处理程序，并将结果传下去

返回Promise

Promise进行错误处理

Promise的所有处理程序：.then,.catch,.finally都是异步的

在JS内部维护着一个内部队列promise jobs，通常被称为微任务队列

只有JS引擎中没有其他任务在执行时，才会执行任务队列中的代码

也就是说，当一个promise准备就绪时，它的处理程序就会被放到这个任务队列中，它们不会立刻被执行，只有当JS引擎中当前代码都执行完毕，才会去队列中获取并执行它们
